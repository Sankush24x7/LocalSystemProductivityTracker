# AI Productivity Tracker (WPF, Offline, Tray-Based)

A privacy-first Windows productivity tracker that runs in background tray mode, tracks activity offline, manages tasks, and generates daily HTML reports.

## Repository Description

AI Productivity Tracker is a lightweight desktop app for developers and professionals who want local-only productivity analytics with task planning, due alerts, and report automation.

## Quick Start

```powershell
dotnet build ProductivityTracker.sln -c Release
cd .\dist\ProductivityTrackerLite
.\ProductivityTracker.App.exe
```

## 12) Settings Reference (`settings.json`)

### `ScreenshotEnabled` (bool)
- Controls whether desktop screenshots are captured in the background.
- `true`: screenshots are saved periodically.
- `false`: screenshot capture is disabled.

### `ScreenshotIntervalMinutes` (int)
- Interval between screenshot captures (in minutes).
- Used only when `ScreenshotEnabled = true`.
- Typical values: `1` to `120`.

### `IdleThresholdMinutes` (int)
- Minutes of no keyboard/mouse input before user state is marked as idle.
- Example: value `3` means after 3 minutes inactive, activity becomes `Idle`.

### `ActivityTrackingIntervalSeconds` (int)
- How frequently activity samples are recorded for timeline/report.
- Supported values in UI: `1, 2, 5, 10, 30, 60`.
- Lower values give finer detail but larger activity JSON files.

### `DueSoundEnabled` (bool)
- Controls sound alert when a task reaches/passes due datetime.
- `true`: system alert sound plays.
- `false`: no sound.

### `DueTrayBalloonEnabled` (bool)
- Controls tray balloon popup when a task reaches/passes due datetime.
- `true`: tray balloon appears.
- `false`: no tray balloon.

### `DueCheckIntervalSeconds` (int)
- Frequency of due-date checks.
- Supported values in UI: `10, 30, 60`.
- Smaller value means faster due-alert reaction.

### `BackupPassword` (string)
- Password used to encrypt/decrypt backup files (`.ptbackup`).
- Required for creating encrypted backups and restoring encrypted backups.

### `MailFrom` (string)
- Sender email id used by SMTP login and mail sender.

### `MailTo` (string)
- Recipient email id for report delivery.

### `MailPassword` (string)
- App password / SMTP password used for authentication.

### `SmtpHost` (string)
- SMTP server host.
- Example: `smtp.gmail.com`.

### `SmtpPort` (int)
- SMTP server port.
- Typical values: `587` (TLS), `465` (SSL).

### `SmtpUseSsl` (bool)
- Enables SSL/TLS for SMTP connection.
- Keep `true` for most providers.

Default due-alert behavior:
- Sound: enabled
- Tray balloon: enabled
- Due check interval: `30` seconds
- Activity tracking interval: `5` seconds

## 15) Task-Based Productivity Score and Points

The dashboard score is now task-based (not activity-only):
- If `Total Tasks = 0`, score is `0%`.
- Each task is categorized as:
  - Good: completed within due datetime
  - Neutral: on-time ongoing or completion with no due-date penalty
  - Bad: overdue task or completed after due datetime

Score formula:
- `Score % = (Good*100 + Neutral*50 + Bad*0) / TotalTasks`
- Final value is clamped to `0..100`.

Task Points block:
- Good task: `+10`
- Neutral task: `+3`
- Bad task: `-8`
- `Task Points = Good*10 + Neutral*3 - Bad*8`

Task row colors in grid:
- Completed: green
- Running/In progress: yellow
- Overdue: red

TAT column:
- Shows due date/time text only (`dd-MMM-yyyy hh:mm tt`)
- No "Overdue ..." label in this column.



## 16) New Advanced Features Added

### A) Pre-Due Task Reminders (5 / 15 / 30 mins)
- Reminders trigger automatically before due time.
- At due time, existing red popup + sound + tray balloon behavior continues.
- Reminder toggles are available in Settings:
  - `Enable 30-minute reminder`
  - `Enable 15-minute reminder`
  - `Enable 5-minute reminder`

### B) One-Click Backup / Restore (JSON + Reports + Screenshots)
- Top dashboard buttons:
  - `Backup` creates an encrypted `.ptbackup` snapshot in `%LocalAppData%\\ProductivityTrackerLite\\Backups`
  - `Restore` lets you pick a backup file and restore local files
- Tray menu also includes:
  - `Show App Shortcuts` (displays global hotkeys)
  - `Backup Now`
  - `Restore Backup`

### C) Pause Tracking Mode (Privacy)
- Use `Pause Tracking` button (or tray menu) to stop activity sampling temporarily.
- While paused:
  - No activity time increments are recorded.
  - App continues running in tray.
- Toggle again to resume.

### D) App/Category Tagging Rules
- Configure in Settings under `Category Rules`.
- Format: one rule per line:
  - `devenv=Coding`
  - `chrome=Browsing`
  - `excel=Other`
- Matching is token-based against active app/window label.

### E) Weekly/Monthly In-App Trend Dashboard
- Click `Trends` (or tray `Open Trends`).
- Shows:
  - 7-day and 30-day active time
  - Day-wise table (active/idle/score)
  - Goal progress bars (hours/tasks)

### F) Data Retention Auto Cleanup
- Settings includes:
  - `Enable auto cleanup of old files`
  - Keep-days for activity JSON, screenshots, reports
- Cleanup runs at startup and after settings save.

### G) Lock/PIN for Dashboard and Reports
- Enable in Settings: `Enable PIN lock for dashboard/report access`
- Set PIN once in `Set PIN` field (stored as SHA-256 hash).
- Protected actions:
  - Open dashboard
  - Open report
  - Open report manager
  - Open trends
- Tray menu has `Lock Now` for immediate lock.

### H) Productivity Goals
- Settings:
  - `Daily active goal (hours)`
  - `Daily task goal (count)`
- Used in in-app trend dashboard progress blocks.

### I) Startup Hardening / Self-Heal
- Startup registration now attempts both:
  - Registry Run key
  - Startup-folder launcher (`.vbs`)
- Main status shows startup self-check result.
- Hotkey auto-recovery timer periodically re-registers global hotkeys.

### J) Offline Update Check (Optional)
- Settings:
  - `Enable offline update check`
  - `Manifest folder/file path`
- App looks for manifest file `productivity-update.json` (if a folder path is provided).
- Tray menu: `Check Offline Update`
- If a newer version is found, app prompts to open local package path.

### K) Health Check Panel
- Dashboard includes a `Health Check` card with:
  - `Tracking: Running/Paused`
  - `Hotkeys: OK/Recovering`
  - `Startup: OK/Needs attention`

### L) Quick Add Task Popup
- Open mini task form quickly using:
  - tray menu `Quick Add Task`
  - hotkey `Ctrl + Shift + N`
- Saves task instantly without opening full editor flow.

### M) Send Report by Email
- Available from dashboard button `Send Report` and tray menu `Send Report`.
- Mail body is HTML report content.
- Attachments include:
  - daily HTML report file
  - daily JSON activity file

## 17) How To Use New Features

### 1. Configure Settings
1. Open dashboard.
2. Click `Settings` from tray menu.
3. Set reminders, retention, category rules, goals, PIN, and offline update path.
4. Click `Save`.

### 2. Use Backup / Restore
1. Click `Backup` to create a zip snapshot.
2. Click `Restore` and select a backup zip to restore files.
3. Dashboard data refreshes automatically after restore.

### 3. Pause / Resume Tracking
1. Click `Pause Tracking` on top bar (or tray).
2. Use `Resume Tracking` to continue.

### 4. Open Trends
1. Click `Trends`.
2. Check 7-day/30-day activity and goal progress.

### 5. Enable PIN Lock
1. Open Settings.
2. Check `Enable PIN lock for dashboard/report access`.
3. Enter PIN in `Set PIN`.
4. Save.
5. Use tray `Lock Now` anytime.

### 6. Configure Offline Update Check
1. Prepare local manifest JSON (example below).
2. Set manifest folder path in Settings.
3. Enable `offline update check`.
4. Use tray `Check Offline Update`.

### 7. Configure Encrypted Backup
1. Open Settings.
2. Set `Backup encryption password`.
3. Save.
4. Use `Backup` / `Backup Now` to generate encrypted `.ptbackup`.
5. Use same password when restoring.

### 8. Configure Send Report Mail
1. Open Settings.
2. Fill `Mail From`, `Mail To`, `Mail Password`.
3. Fill `SMTP Host`, `SMTP Port`, SSL toggle.
4. Save.
5. Click `Send Report` from dashboard or tray.

Example manifest (`productivity-update.json`):

```json
{
  "version": "1.2.0",
  "packagePath": "D:\\OfflineUpdates\\ProductivityTracker.App.exe",
  "notes": "Bug fixes and performance improvements"
}
```
